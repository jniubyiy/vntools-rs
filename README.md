# 🦀 VNTools-RS – форк на Rust для извлечения игровых ресурсов

[![Rust](https://img.shields.io/badge/Rust-1.70+-orange.svg)](https://www.rust-lang.org/)
[![License](https://img.shields.io/badge/License-GPLv2-blue.svg)](LICENSE)

**VNTools-RS** — это форк оригинального проекта [vntools](https://github.com/hiirotsuki/vntools) (автор: **hiirotsuki**). Оригинальный инструментарий написан на C и предназначен для распаковки и конвертации ресурсов из различных игровых форматов (визуальные новеллы, RPG, и т.д.).  
Данный форк полностью переписан на **Rust** с целью упростить сборку, повысить безопасность и удобство использования, а также предоставить набор отдельных исполняемых файлов для каждого формата.

**Главное преимущество**: все утилиты работают по принципу **«перетащил и забыл»** – просто перетащите файл нужного формата на соответствующий `.exe`, и он будет распакован или сконвертирован в папке с исходным файлом.

## 🖱️ Как пользоваться

### Базовое использование (для большинства утилит)
1. **Перетащите** один или несколько файлов нужного формата на соответствующий `.exe` файл.
2. Программа обработает все переданные файлы, создав рядом с каждым результат (конвертированное изображение или папку с распакованными файлами).
3. В консоли (если она открыта) будет выводиться прогресс и сообщения об ошибках.

### Особенности для некоторых утилит
- **`nekopunch_unpack.exe`** – требует указания режима имён (32-bit или 64-bit):  
  `nekopunch_unpack.exe -name32 archive.pak`  
  `nekopunch_unpack.exe -name64 archive.pak`  
  (обязательно укажите корректный режим, иначе распаковка не сработает).

- **`ggp2png.exe`, `cgd2bmp.exe`, `ng3_unpack.exe`, `hiz_unpack.exe`** – просто перетащите файл(ы), результат появится рядом с тем же именем, но с расширением `.png` или `.bmp`.

### Пример
```cmd
# Распаковать архив PAK из игры Cromwell
pak_unpack.exe graphic.pak

# Конвертировать все GGP-файлы в папке (перетащите их все сразу)
ggp2png.exe *.ggp
```

## 🔧 Сборка из исходников (для разработчиков)

Если вы хотите собрать утилиты самостоятельно (например, для других ОС):

1. Установите [Rust](https://rustup.rs/) (стабильная версия).
2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/vntools-rs.git
   cd vntools-rs
   ```
3. Соберите все утилиты:
   ```bash
   cargo build --release
   ```
4. Готовые исполняемые файлы появятся в папке `target/release/`.

Для статической сборки под Windows (без внешних DLL) используется флаг `+crt-static`, уже прописанный в корневом `Cargo.toml`.
